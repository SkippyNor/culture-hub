A "named slice" is a custom collection that you compose yourself by selecting slices or parts (through use of query terms) from either a single or multiple collections.

In order to give end-users access to a named-slice you first need to create a CMS page in which the slice will be embedded. Important to note is that when creating this CMS page, is that you DO NOT attach the cms page to a menu (for the menu options of the CMS page choose "None"). Also make sure to publish the page.

Once you have done this you can start creating your named slice. When it comes to choosing the CMS page for your named slice, choose the one you just created. After you save the named slice a link will appear in the main menu with the name you entered for the slice. When you click this link you will go to the CMS page you created and from there you can search through the custom collection (named slice) you just created.
